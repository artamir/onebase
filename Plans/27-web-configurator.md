# Этап 27 — Веб-конфигуратор (редактирование метаданных в браузере)

**Статус:** ⬜ Не начато

## Контекст

Сейчас конфигурация редактируется только через YAML-файлы на диске (или через конфигуратор в лаунчере, который показывает дерево — но не позволяет менять файлы из браузера). Веб-конфигуратор даёт возможность добавлять поля, создавать объекты и изменять схему прямо в браузере, без выхода из пользовательского режима.

## UX

Новый раздел **Конфигуратор** в левом меню (только для администраторов):

### Главная страница

Дерево метаданных с кнопкой «+ Создать» рядом с каждым типом объекта.

### Редактор справочника / документа

```
Справочник: Номенклатура
[Иерархический ☑]

Поля:
┌──────────────┬────────────────────────┬───┐
│ Наименование │ string                 │ ✕ │
│ Артикул      │ string                 │ ✕ │
│ Цена         │ number                 │ ✕ │
│ Категория    │ reference:Категория    │ ✕ │
└──────────────┴────────────────────────┴───┘
[+ Добавить поле]

[Сохранить]  [Удалить объект]
```

При сохранении:
1. YAML-файл перезаписывается на диске (file-режим) или в `_onebase_config` (database-режим)
2. `onebase migrate` выполняется автоматически (`ALTER TABLE ADD COLUMN IF NOT EXISTS`)
3. Страница обновляется

### Редактор перечисления

Список значений с возможностью добавлять / удалять / менять порядок drag-and-drop.

### Редактор отчёта

Текстовое поле для запроса (с подсветкой SQL), список параметров.

## Изменения в коде

**`internal/ui/configurator_web.go`** (новый файл):
- `GET /configurator/` — главная страница конфигуратора
- `GET /configurator/catalog/{name}` — форма редактирования справочника
- `POST /configurator/catalog/{name}` — сохранить изменения
- `POST /configurator/catalog` — создать новый справочник
- `DELETE /configurator/catalog/{name}` — удалить (с подтверждением)
- Аналогично для documents, enums, registers, reports

**`internal/project/writer.go`** (новый файл):
- `WriteEntityYAML(dir string, entity *metadata.Entity) error`
- `DeleteEntityYAML(dir string, name string) error`
- Для database-режима — `SaveEntityToConfigDB(ctx, db, entity)`

**`internal/ui/templates.go`**:
- Добавить страницы конфигуратора (формы с полями, drag-and-drop список перечислений)

**`internal/ui/server.go`**:
- Маунт `/configurator/` (только для `isAdmin`)

## Ограничения

- Удаление поля из YAML не удаляет колонку из БД (как и сейчас). Предупреждение в UI.
- Переименование поля — создаёт новое поле, старое остаётся. Требует ручной миграции.
- Только для `config_source: file` (для database-режима — в следующей итерации).

## Тесты

- POST `/configurator/catalog/X` с новым полем → YAML обновлён на диске + колонка создана в БД
- Создание нового справочника → файл создан → объект виден в UI
